# aisdk-compact

Context compaction middleware for the [Vercel AI SDK](https://ai-sdk.dev). Automatically summarizes older messages when your conversation approaches the model's context window limit.

## Install

```bash
npm install aisdk-compact
# or
bun add aisdk-compact
```

Requires `ai@^6` as a peer dependency.

## Usage

Wrap any model with `compactMiddleware` using `wrapLanguageModel`:

```typescript
import { generateText, wrapLanguageModel, gateway } from "ai";
import { compactMiddleware } from "aisdk-compact";

const model = wrapLanguageModel({
  model: gateway("anthropic/claude-sonnet-4"),
  middleware: compactMiddleware({
    threshold: 0.8, // compact at 80% of context window (default)
  }),
});

// Use normally — compaction happens transparently
const { text } = await generateText({
  model,
  messages: longConversation,
});
```

The middleware intercepts every call. When the estimated token count exceeds the threshold, it:

1. Preserves all system messages
2. Keeps the most recent messages intact
3. Summarizes older messages into a single compact message via the model
4. Forwards the compacted conversation to the model

## Options

```typescript
compactMiddleware({
  // Context window size in tokens.
  // Auto-resolved from model ID if omitted (supports OpenAI, Anthropic, Google, Meta, Mistral, etc.)
  maxTokens: 128_000,

  // Fraction of maxTokens that triggers compaction (default: 0.8)
  threshold: 0.8,

  // Number of recent messages to always preserve (default: 6)
  recentMessageCount: 6,

  // Use a cheaper/faster model for generating summaries
  compactionModel: gateway("anthropic/claude-haiku-4"),

  // Custom system prompt for the summarization call
  compactionPrompt: "Summarize this conversation...",

  // Custom token estimation function (default: JSON char length / 4)
  estimateTokens: (messages) => myTokenCounter(messages),

  // Callback fired after compaction occurs
  onCompaction: ({ originalTokens, compactedTokens, removedMessageCount }) => {
    console.log(`Compacted ${removedMessageCount} messages: ${originalTokens} -> ${compactedTokens} tokens`);
  },
});
```

## Auto-resolved context sizes

When `maxTokens` is omitted, the middleware looks up the context window from the model ID. Supported models include:

| Provider | Models | Context |
|----------|--------|---------|
| OpenAI | gpt-4.1 | 1M |
| OpenAI | gpt-4o, gpt-4-turbo | 128K |
| OpenAI | o1, o3, o4-mini | 200K |
| Anthropic | claude-3.x, claude-4.x | 200K |
| Google | gemini-2.x | 1M |
| Google | gemini-1.5-pro | 2M |
| Meta | llama-3.1+ | 128K |
| Mistral | mistral-large, mistral-small | 128K |
| DeepSeek | deepseek | 128K |

For unrecognized models, the middleware logs a warning and defaults to 60K tokens. Pass `maxTokens` explicitly to avoid this.

## Using a separate compaction model

To reduce cost, use a smaller model for generating summaries:

```typescript
import { wrapLanguageModel, gateway } from "ai";

const model = wrapLanguageModel({
  model: gateway("anthropic/claude-sonnet-4"),
  middleware: compactMiddleware({
    compactionModel: gateway("anthropic/claude-haiku-4"),
  }),
});
```

## How it works

The middleware uses the AI SDK's `transformParams` hook to intercept calls before they reach the model. When compaction triggers:

```
Before:  [system] [msg1] [msg2] ... [msg48] [msg49] [msg50]
                   ^^^^^^^^^^^^^^^^^^^^^^^^
                   older messages → summarized

After:   [system] [summary] [msg47] [msg48] [msg49] [msg50]
                             ^^^^^^^^^^^^^^^^^^^^^^^^^^^^
                             recent messages preserved
```

The summary is generated by calling `model.doGenerate()` on the underlying model (or a separate `compactionModel`) with a summarization prompt. The summary preserves key facts, decisions, code context, and unresolved items.
